## 프로젝트 개요

brelio는 경제 뉴스를 더 쉽게 읽고 학습 기록으로 남기기 위한 웹 서비스입니다. 프론트엔드와 백엔드를 함께 다루며, 콘텐츠 조회와 학습 트래킹 흐름을 연결하는 것을 목표로 합니다.

## 문제 정의

경제 뉴스는 용어와 맥락을 함께 이해해야 지속적으로 학습하기 쉽습니다. 단순 뉴스 목록을 넘어 사용자가 읽은 내용, 요약, 학습 상태를 다시 확인할 수 있는 구조가 필요합니다.

## 내가 맡은 역할

개인 프로젝트로 Full-stack 역할을 맡았습니다. 프론트엔드 화면, API 서버, 데이터베이스, 배포 환경을 함께 다뤘으며, 상세 내용은 실제 구현 범위에 맞춰 계속 정리할 예정입니다.

## 핵심 구현

- Vite와 React 기반 웹 클라이언트 구성
- TanStack Query를 활용한 뉴스/학습 데이터 요청 상태 관리
- Express, Prisma, PostgreSQL 기반 API와 데이터 모델 구성
- Docker와 GCP Cloud Run 배포 흐름 검토

## 기술 선택 이유

Vite는 빠른 개발 경험을 제공하고, Express와 Prisma는 초기 백엔드 구조를 명확하게 만들기 좋습니다. PostgreSQL은 관계형 데이터가 필요한 학습 기록과 콘텐츠 메타데이터를 다루기에 적합합니다.

## 트러블슈팅

구체적인 사례는 배포, 크롤링/요약 흐름, 데이터 모델, 인증 또는 관리자 기능 중 실제로 해결한 문제를 중심으로 추가합니다.

## 결과

현재는 프로젝트의 방향과 기술 구성을 보여주는 초안입니다. 실제 화면과 운영 방식, 개선 이력을 추가하면 케이스 스터디로 확장할 수 있습니다.

## 앞으로 개선할 점

- 핵심 사용자 흐름을 기준으로 상세 설명 보강
- API와 데이터 모델 다이어그램 추가
- 배포 및 운영 중 해결한 문제 정리
