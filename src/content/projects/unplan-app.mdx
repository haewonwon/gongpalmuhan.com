## 프로젝트 개요

unplan-app은 사용자의 일정과 선호를 바탕으로 더 유연한 계획 수립을 돕는 모바일 앱입니다. 현재 내용은 프로젝트를 설명하기 위한 초안이며, 실제 기능 단위의 구현 기록은 추후 보완할 예정입니다.

## 문제 정의

스마트 스케줄러는 단순한 일정 목록보다 사용자의 맥락, 우선순위, 변경 가능성을 함께 다뤄야 합니다. 모바일 환경에서는 입력 흐름이 길어지지 않도록 화면 전환과 상태 관리가 명확해야 합니다.

## 내가 맡은 역할

Frontend 역할로 참여하며 React Native와 Expo 기반 화면 구현, API 연동 구조, 클라이언트 상태 관리 방식을 정리하는 방향으로 기여했습니다. 팀 프로젝트이므로 전체 제품 구현을 혼자 담당한 것은 아니며, 담당 범위 중심으로 내용을 계속 업데이트할 예정입니다.

## 핵심 구현

- Expo 기반 앱 구조에서 화면 단위 컴포넌트를 정리
- TanStack Query와 Axios를 활용한 서버 상태 연동 흐름 검토
- Orval 기반 API 클라이언트 생성 구조를 프론트엔드 코드와 연결
- 일정 생성, 조회, 수정 흐름에서 필요한 상태 분리 기준 정리

## 기술 선택 이유

React Native와 Expo는 모바일 앱을 빠르게 검증하기에 적합합니다. TanStack Query는 서버 상태의 캐싱, 로딩, 에러 상태를 일관되게 다루기 위해 선택했고, Zustand는 화면 간 공유가 필요한 클라이언트 상태를 가볍게 관리하는 데 적합하다고 판단했습니다.

## 트러블슈팅

초안 단계에서는 구체적인 장애 사례를 확정하지 않았습니다. 이후 API 스펙 변경, 인증 흐름, 빌드 설정, 상태 동기화 이슈 중 실제로 해결한 사례를 중심으로 정리할 예정입니다.

## 결과

현재는 포트폴리오 서술 구조를 먼저 마련한 상태입니다. 실제 화면, PR, 구현 범위가 정리되는 대로 결과 섹션을 사실 기반으로 업데이트합니다.

## 앞으로 개선할 점

- 실제 담당 화면과 기능 단위로 내용 구체화
- API 연동과 상태 관리 구조를 코드 예시와 함께 정리
- 팀 내 협업 방식과 리뷰 과정 보완
